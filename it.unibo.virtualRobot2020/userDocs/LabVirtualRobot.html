INTRODUCTION

Our company (Unibo) has developed a virtual environment (named WEnv) that includes a virtual robot.
The virtual environment has been mainly built by Pierfrancesco Soffritti using the three.js JavaScript library (see ConfigurableThreejsApp ) .

Starting

The reference project for this introduction is it.unibo.robots19.
  1. Install Node.js
  2. Open a terminal on it.unibo.robots19\node\WEnv\server and execute 
    npm install
  3. In the directory it.unibo.robots19\node\WEnv\WebGLScene execute 
    npm install
  4. In the directory it.unibo.robots19\node\WEnv\server\src execute 
    node main 8999
  5. Open a browser on 
    localhost:8090

INTERACTION

The virtual robot uses the Node library https://github.com/einaros/ws to accept commands sent on a websocket at port 8999.
The commands are described in the file README.md. For example: 
moveForward :  `{ "type": "moveForward", "arg": 300 }`
moveBackward : `{ "type": "moveBackward", "arg": 300 }`
 
turnRight :    `{ "type": "turnRight", "arg": 300 }`
turnLeft :     `{ "type": "turnLeft", "arg": 300 }` 

stop :         `{ "type": "alarm" }`
Moreover, the virtual environment sends to the clients connected on the ws port 8091:
  • Data emitted by the sonars included in the scene when they detect some object (the moving robot)
  • Data emitted by the impact sensors put in front and in the rear of the robot, when they detect an obstacle (fixed or mobile) 
{ "sonarName": "sonarName", "distance": 1, "axis": "x" }

{ "collision" : "false", "move": "moveForward"}

USING THE VIRTUAL ROBOT

Examples of clients are reported in project it.unibo.virtualrobotclient

Clients written in Node.js

wsclientToWenv.js The client first moves forward and backward. Afterwards, it works as an observer that:
  • show information emitted by some sonar
  • rings a bell if the virtual robot collides with some virtual obstacle

The client interacts with the WEnv via web-scokets, by using the WebSocket library for Node.js.
axiosclientToWenv.js The client walks along the boundary of the room.

The client interacts via HTTP with the WEnv by using the axios library for Node.js.
boundaryReactiveWebsocket.js The client walks along the boundary of the room in a event-driven way

The client interacts with the WEnv via web-scokets, by using the websocket library for Node.js.
wsBoundaryReactive.js The client walks along the boundary of the room in a event-driven way

The client interacts with the WEnv via web-scokets, by using the ws library for Node.js see https://github.com/einaros/ws

Clients written in Java

ClientUsingPost.java The client walks along the boundary of the room.

It exploits the library org.apache.http.client (see also HttpClient - Tutorial) to implement HTTP requests.

The handling of JSON answers is done using the library javax.json.stream.JsonParser or the library org.json.simple.parser (see also JSON.simple - Quick Guide)
ClientWebsockJavax.java The client walks along the boundary of the room.

The client interacts with the WEnv via web-scokets, by using the javax.websocket library .

WebSocket server e client per node.js. Una delle librerie più veloci se non la più veloce. This module does not work in the browser. Browser clients must use the native WebSocket object.

Docker

gradle build                            Create the distribution
docker build -t virtualrobotdisi:1.0 .  Create the image using the Dockerfile

docker run -ti -p 8090:8090 -p 8091:8091 --rm  virtualrobotdisi:1.0 /bin/sh	local test

Tag the image
docker tag virtualrobotdisi:1.0 natbodocker/virtualrobotdisi:1.0

Login in DockerHub (get the password from a private file)
type docker_password.txt | docker login --username natbodocker --password-stdin

Register the image on DockerHub
docker push natbodocker/virtualrobotdisi:1.0

Use docker-compose.yaml
docker-compose up -d  or docker-compose run -d --service-ports wenv
